# 料理動画→レシピ変換アプリ 総合調査レポート

## 結論: 技術的に実現可能、日本市場にブルーオーシャンあり、個人開発でも黒字化のハードルは低い

---

## 1. 技術面: どう作るか

### 処理パイプライン

```
[動画入力]
    → [音声抽出] FFmpeg
    → [音声認識] OpenAI GPT-4o Mini Transcribe ($0.003/分)
    → [動画解析] Gemini 2.5 Flash (ネイティブ動画対応, $0.30/M tokens)
    → [OCR] PaddleOCR (無料・セルフホスト)
    → [レシピ構造化] GPT-4o mini (JSON Mode)
    → [構造化レシピJSON]
```

### ターゲットユーザー

1. **SNS料理動画ヘビーユーザー（20-30代女性）**
   - YouTube/TikTok/Instagramで料理動画を日常的に視聴
   - 動画をスクショして自分でメモを取っている

2. **海外料理動画を参考にする日本のユーザー**
   - 言語の壁があり、英語の料理動画からレシピを正確に把握できない

3. **料理初心者・新婚・一人暮らし開始者**
   - 動画で見て「作りたい」が、材料の分量や手順に不安

4. **忙しい共働き家庭の親（30-40代）**
   - 献立を考える時間がない

### ユーザーのペインポイント

- 動画を見ながら料理すると手が汚れて再生操作が困難
- 材料・分量を確認するために何度も巻き戻す手間
- お気に入り動画をブックマークしても、レシピを探し出せない
- 海外動画の食材が日本で手に入らない（代替品が不明）
- 動画の分量が家族の人数に合わない

### 差別化ポイント

| ポイント | 説明 |
|---------|------|
| 日本語ネイティブ対応 | 和食特有の食材・調理法（落とし蓋、面取り等）に対応 |
| 食材ローカライズ | 海外食材→日本で入手可能な代替品を自動提案 |
| 人数調整・アレルギー対応 | 分量の自動スケーリング、アレルゲン自動検出 |
| 一気通貫UX | 動画→レシピ→献立→買い物リスト |
| オフラインレシピカード | キッチンでオフライン閲覧可能 |
| 料理YouTuberコラボ | リュウジ等の人気チャンネルと連携 |

### 日本市場特有の考慮点

- 動画レシピへのシフトが急速（クラシルの成長が証明）
- LINEとの親和性が高い（キユーピー「レシピトーク」がLINEで成功）
- 「管理栄養士監修」等のお墨付きが重要
- YouTube料理チャンネルのコメント欄で「レシピをテキストにしてほしい」要望が多い

## 4. リスクと対策

| リスク | 影響度 | 対策 |
|--------|--------|------|
| YouTube/TikTokが同機能を内蔵 | 高 | 単なる変換ツールではなく「献立+買い物」の総合アプリに |
| クラシルが同機能を追加 | 中 | 先行者優位を確保、ニッチ（外部動画変換）に特化 |
| 著作権・利用規約の懸念 | 中 | ユーザー自身のアップロード方式を基本に。YouTube Data APIで字幕のみ取得 |
| 日本語の料理用語認識精度 | 低 | カスタム語彙・ファインチューニングで改善 |
| API費用の高騰 | 低 | 複数プロバイダーの併用でリスク分散 |

## 5. 推奨アクションプラン

### Phase 1: MVP（0〜1ヶ月）
- YouTube URLまたは動画アップロード → レシピ変換のコア機能のみ
- Next.js Webアプリとして構築（PWA対応）
- Free + Proの2プランでシンプルに開始
- RevenueCat導入（無料枠で十分）

### Phase 2: ローンチ（1〜3ヶ月）
- Product Hunt + TikTokでローンチ
- ユーザーフィードバックで改善
- 初期100人の有料ユーザー獲得目標
- MRR 5万円（黒字化）目標

### Phase 3: 成長（3〜12ヶ月）
- モバイルアプリ（React Native）リリース
- Basicプラン追加（3段階化）
- 献立管理・買い物リスト機能追加
- 食材アフィリエイト連携開始
- マイクロインフルエンサー施策
- MRR 30万円目標

### Phase 4: スケール（12ヶ月〜）
- API提供（B2B）開始
- 海外展開（英語版）
- 年額プラン・ファミリープランの導入
- MRR 100万円以上目標
