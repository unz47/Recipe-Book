# UI デザインルール

## デザイン原則

**モダン・ミニマル**（Notion / Linear 風の余白を活かしたクリーンなデザイン）

1. **余白を恐れない** — 情報密度より視認性を優先する。コンテンツに十分な余白を与え、呼吸感のあるレイアウトにする
2. **一貫性** — 同じ要素は同じ見た目と振る舞いで表現する。shadcn/ui のデフォルトスタイルを尊重する
3. **コンテンツファースト** — UIの装飾は最小限に。写真・テキストなどのコンテンツが主役になるように設計する
4. **階層の明確化** — フォントサイズ・ウェイト・カラーの変化で情報の優先度を伝える。影やボーダーの多用は避ける

## タイポグラフィ

### フォントファミリー

```css
--font-sans: "Inter", "Noto Sans JP", system-ui, sans-serif;
--font-mono: "JetBrains Mono", monospace;
```

- 欧文: Inter
- 和文: Noto Sans JP
- コード: JetBrains Mono

### フォントサイズスケール

| 用途 | サイズ | Tailwind クラス | ウェイト |
|------|--------|----------------|---------|
| ページタイトル | 30px | `text-3xl` | `font-bold` (700) |
| セクション見出し | 24px | `text-2xl` | `font-semibold` (600) |
| カード見出し | 20px | `text-xl` | `font-semibold` (600) |
| サブ見出し | 16px | `text-base` | `font-medium` (500) |
| 本文 | 14px | `text-sm` | `font-normal` (400) |
| 補足テキスト | 12px | `text-xs` | `font-normal` (400) |

### 行間

- 見出し: `leading-tight` (1.25)
- 本文: `leading-relaxed` (1.625)
- 補足: `leading-normal` (1.5)

## スペーシング・レイアウト

### スペーシングスケール

Tailwind CSS のデフォルトスケールに従い、以下を基本とする。

| 用途 | サイズ | Tailwind |
|------|--------|----------|
| 要素内パディング（小） | 8px | `p-2` |
| 要素内パディング（中） | 12px | `p-3` |
| 要素内パディング（大） | 16px | `p-4` |
| カード内パディング | 24px | `p-6` |
| セクション間マージン | 32px | `my-8` |
| ページ余白（モバイル） | 16px | `px-4` |
| ページ余白（デスクトップ） | 24px-32px | `px-6` ~ `px-8` |

### レイアウトの最大幅

| 用途 | 幅 | Tailwind |
|------|-----|----------|
| コンテンツ（読み物） | 672px | `max-w-2xl` |
| コンテンツ（標準） | 896px | `max-w-4xl` |
| アプリケーション全体 | 1152px | `max-w-6xl` |

### グリッド

- レシピ一覧: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-6`
- フォームレイアウト: 1カラム、`max-w-lg`

## コンポーネント設計ガイドライン

### カード

```
角丸: rounded-xl (12px)
ボーダー: border border-border
シャドウ: なし（フラットデザイン）、ホバー時に shadow-sm
背景: bg-card
パディング: p-6
```

### ボタン

shadcn/ui の Button コンポーネントをそのまま使用する。

| バリアント | 用途 |
|-----------|------|
| `default` | 主要アクション（保存、送信） |
| `secondary` | 補助アクション |
| `outline` | キャンセルなどの補助操作 |
| `ghost` | ナビゲーション、ツールバーのアクション |
| `destructive` | 削除系アクション |

サイズ: `default` を基本とし、コンテキストに応じて `sm` / `lg` を使い分ける。

### 入力フォーム

```
角丸: rounded-lg (8px)
ボーダー: border border-input
フォーカス: ring-2 ring-ring
高さ: h-10 (default), h-9 (sm)
```

- ラベルは入力フィールドの上に配置する
- プレースホルダーはヒントとして使い、ラベルの代わりにしない
- エラーメッセージはフィールド直下に `text-destructive text-xs` で表示する

### アイコン

- Lucide React を使用する
- サイズ: 16px (`size={16}`) を基本とする
- ボタンアイコン: テキストの左に配置、`mr-2` のギャップ
- 単独アイコンボタン: `size="icon"` を使用する

## レスポンシブ対応ルール

### ブレークポイント

Tailwind CSS のデフォルトブレークポイントを使用する。

| プレフィックス | 幅 | 対象 |
|--------------|------|------|
| （なし） | 0px~ | モバイル |
| `sm` | 640px~ | 大きめのモバイル |
| `md` | 768px~ | タブレット |
| `lg` | 1024px~ | デスクトップ |
| `xl` | 1280px~ | 大画面 |

### モバイルファースト

- スタイルはモバイルから記述し、大きな画面サイズに向けて拡張する
- ナビゲーション: モバイルではボトムバー、デスクトップではサイドバー
- タッチターゲット: 最小 44x44px を確保する

## アニメーション・トランジション

### 基本方針

- アニメーションは控えめに。目的のある動きのみ実装する
- ユーザーの操作に対するフィードバックとして使う
- `prefers-reduced-motion` を必ず考慮する

### トランジション設定

| 用途 | プロパティ | 時間 | イージング |
|------|----------|------|----------|
| ホバー・フォーカス | color, background, border, shadow | 150ms | `ease-in-out` |
| モーダル・ドロワー | opacity, transform | 200ms | `ease-out` |
| ページ遷移 | opacity | 150ms | `ease-in-out` |

### Tailwind クラス

```
ホバー系: transition-colors duration-150
表示切替: transition-opacity duration-200
変形系: transition-transform duration-200
```

### 使ってよいアニメーション

- フェードイン/アウト
- スライドイン（モーダル、ドロワー）
- スケール（ボタンのホバー: `hover:scale-[1.02]` 程度）

### 使わないアニメーション

- バウンス、スピン（ローディングスピナー除く）
- 過度なパララックス
- 自動再生のアニメーション
